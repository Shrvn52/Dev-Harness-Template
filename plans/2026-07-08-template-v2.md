# Template v2 — structure, staleness, and identity hardening

**Date:** 2026-07-08
**Status:** decided with the operator, in execution
**Origin:** fresh-eyes template review → four identity decisions made explicitly by the
operator (structure, staleness posture, stack posture, keep/cut) + seven defects that
fail the repo's own filter and need no decision.

## The decision filter (write-up of the operator session)

The template's purpose: **a reference implementation of agentic-dev guardrails — the
smallest working app that can carry a complete, copyable harness.** The harness is the
product; the stack is demo material. Every change below passed three questions:

1. **Harness or stack?** Harness → invest. Stack → minimal, swappable-by-deletion, never abstracted.
2. **Does it enforce itself?** A convention that isn't mechanical is a liability.
3. **Does it age?** Anything pinned needs a caretaker — for a _template_, that caretaker
   is a documented routine, not a bot (consumers don't inherit bot installs; a
   `renovate.json` in a template is a promise the cloner's org may never honor).

## Operator decisions (do not relitigate)

- **Structure → npm workspaces.** Kills the root-devDep mirror of backend deps (already
  drifted: root pinned `hono@4.12.25` while backend declared `^4.12.14` — tests were
  type-checked against one resolution and run against another), collapses three lockfiles
  to one, one `npm ci`.
- **Staleness → no automation.** Floor-only `engines` (`>=20`, no upper bound) so future
  Node doesn't hard-fail fresh clones; `.nvmrc` stays the tested pin; the scheduled audit
  lane stays the alarm; a documented **freshness routine** replaces Renovate/Dependabot.
- **Stack → document the swap seams.** Keep Hono + better-sqlite3 + React as the one
  working demo. Add a swap map: which files are harness (survive any stack) vs stack
  (replace wholesale), and what each guardrail needs when a layer is swapped. Rejected:
  ports-and-adapters abstraction (the all-things-to-all-men trap).
- **Keep:** Prettier — but _enforced_ (config-without-enforcement violated filter #2).
  `.archon/` stays (operator uses Archon; the docs already tell non-users to delete it).

## Cluster A — npm workspaces conversion (large)

- Root `package.json`: `"workspaces": ["backend", "frontend"]`; **delete** the devDep
  mirror (`hono`, `@hono/node-server`, `@hono/zod-validator`, `zod`,
  `@types/better-sqlite3`) — `tests/` now resolves them from the hoisted install, so the
  root typecheck and the backend runtime can no longer disagree about versions.
- Delete `backend/package-lock.json` + `frontend/package-lock.json`; regenerate the single
  root lockfile. Root scripts move from `cd`-delegation to `npm run <s> -w <pkg>` where a
  workspace owns the script.
- `shared/` stays a bare dir with its `{"type":"module"}` marker (NOT a workspace — it has
  no deps and must keep its NodeNext emit semantics; see CLAUDE.md).
- `engines` → `">=20"` in all three package.jsons; drop the npm bound.
- Gate: `npm ci` (once) → lint, typecheck, build, `npm test`, `test:smoke:dist` all green.

## Cluster B — CI for workspaces + the path-filter hole

- `pr.yml`: single `npm ci`; drop every `--prefix` install; `cache-dependency-path` →
  root lockfile everywhere; lockfile-drift job checks ONE lockfile.
- **Close the tsconfig gap:** add `tsconfig*.json`, `backend/tsconfig.json`,
  `frontend/tsconfig*.json` to the `lint` path filter (the typecheck lane keys off it) —
  today a PR loosening `strict` skips typecheck entirely.
- Checksum-verify the gitleaks tarball (pinned version + pinned sha256; refresh both via
  the freshness routine).
- `audit.yml`: one `npm audit` at root (workspaces-aware) replaces the three `--prefix` runs.

## Cluster C — tier-1 defects

- Remove unused `react-router-dom` from `frontend/package.json` (never imported).
- `.claude/settings.json` hook `timeout: 30000` → `30` (the field is seconds).
- `tools/dev.mjs`: propagate a crashed child's exit code (today exits 0 on child crash);
  Ctrl-C still exits 0.
- Ratchet test scans `frontend/src` + `shared` + `tests`, not just `backend/src` — makes
  CLAUDE.md's "debt only shrinks" claim true at its stated scope.
- Registry reverse-coverage test strips `//` and `/* */` comments before regex-matching
  import specifiers (a commented-out import currently passes).

## Cluster D — prettier, enforced

- `prettier` devDep at root; `.prettierignore` (lockfile, dist, node_modules, plans/archive);
  `format` + `format:check` scripts; `format:check` joins the CI lint lane.
- One mechanical `prettier --write` commit, separate from all logic changes.

## Cluster E — docs

- `docs/SWAPPING.md` — the swap map (harness vs stack, per-layer instructions, which arch
  tests/config each swap touches). Linked from README + CLAUDE.md.
- Freshness routine in `docs/SETUP.md`: quarterly-ish — run the audit workflow, bump
  `.nvmrc` to current LTS, refresh the gitleaks pin+sha, `npm outdated` sweep.
- Sync CLAUDE.md (command table, workspaces wording replaces "cd-delegation", ratchet
  scope), README, TESTING.md.
